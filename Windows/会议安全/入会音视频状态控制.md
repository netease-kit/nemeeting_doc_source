本文详细介绍如何在Windows平台上使用网易会议组件 NEMeetingKit 配置入会控制功能。您可以设置其他成员入会时的默认音视频状态（如是否自动静音或关闭摄像头）。

## 前提条件

开始本文操作前，请确保您已经完成了 [登录鉴权](https://doc.yunxin.163.com/meeting/guide/jU0MTcwNDY?platform=windows) 的相关配置。

## 方式一：服务端控制入会

通过调用服务端 API 创建会议时，可以设置参会者入会的音视频控制选项。请参考服务端 API 文档 [创建会议](https://doc.yunxin.163.com/meeting/server-apis/TUwMzc3NzY?platform=server) 中的 `securityCtrl` 字段说明。

## 方式二：客户端控制入会

会议主持人在客户端创建会议时，可以通过 [`NEStartMeetingParams.controls`](https://doc.yunxin.163.com/docs/interface/meetingkit/windows/doxygen/Latest/zh/class_n_e_start_meeting_params.html#af0f582a9de4b52ba9b53f404abb8ea1f) 设置参会者入会控制选项。

**入会控制类型说明**

[`NEAttendeeOffType`](https://doc.yunxin.163.com/docs/interface/meetingkit/windows/doxygen/Latest/zh/kit__define__meeting_8h.html#a3721e308d85733c71776c68cf04b1897) 枚举提供以下控制选项：

- **`kAttendeeOffAllowSelfOn`**：入会时关闭，但允许参会者自行开启。
- **`kAttendeeOffNotAllowSelfOn`**：入会时关闭，且不允许参会者自行开启（仅主持人可开启）。
- **`kAttendeeOffNone`**：不做任何控制。

**示例代码如下：**

```C++
// 创建会议开始参数
NEStartMeetingParams startParams;

// 配置会议控制项
std::list<NEMeetingControl> controls;

// 入会时麦克风静音，且不允许自主打开
NEMeetingControl audioControl{kAttendeeOffNone, "audio"};
controls.push_back(audioControl);

// 入会时摄像头关闭，且不允许自主打开
NEMeetingControl currentVideoControl{kAttendeeOffNone, "video"};
videoControl.setAttendeeOff(kAttendeeOffNotAllowSelfOn);
controls.push_back(videoControl);

// 将控制项设置到会议参数
startParams.controls = controls;
startParams.displayName = "主持人";

// 使用设置好的参数开始会议
m_meetingService->startMeeting(param, opts, [this](MeetingErrorCode errorCode, const std::string& errorMessage) {
});
```
