本文介绍如何通过网易会议组件的巡检者角色对会议内容进行巡检，保障会议服务的安全。

## 功能介绍

如何有效地检查和管理会议中可能会存在的违规内容或其他违规行为一直是一个令会议服务提供商头疼的问题。

网易会议组件提供巡检者模式，您可自行搭建巡检页面，通过巡检者的身份加入到会议中进行人工巡查，检查会议中是否存在违规内容或其他违规行为。如果发现了违规内容，您可以通过 [结束会议](https://doc.yunxin.163.com/meeting/server-apis/Dk3MDU5NTc?platform=server) REST 接口结束会议，从而保障整体会议服务的安全。

## 准备工作

根据本文操作前，请确保您已经完成了以下设置：

1. 已实现音视频会议，具体请参考 [跑通示例项目](https://doc.yunxin.163.com/meeting/guide/TUxNDEzMTI?platform=web) 或 [实现基础功能（Web）](https://doc.yunxin.163.com/meeting/guide/zE2MDY2ODU?platform=web)。

2. 已接入 [会议抄送](https://doc.yunxin.163.com/meeting/server-apis/Dc2Mjk1MDE?platform=server)，根据抄送事件获取当前进行的会议、会议开始和结束的状态、会议号（meetingNum）。

3. 请自行搭建巡检界面，以便可视化巡检。

## 配置步骤

1. 给某个房间的巡检者角色绑定用户，具体请参考 [增加巡检者](https://doc.yunxin.163.com/meeting/server-apis/jM2NDU1NTI)。

2. 巡检者加入会议。

    巡检者调用 `join` 方法加入会议，具体请参考 [加入会议](https://doc.yunxin.163.com/meeting/guide/zE2MDY2ODU?platform=web#%E5%8A%A0%E5%85%A5%E4%BC%9A%E8%AE%AE)。

    **示例代码** 如下：

    ```JavaScript
    // 加入会议
    const neMeetingKit = NEMeetingKit.getInstance()
    const meetingService = neMeetingKit.getMeetingService()

    const param = {
      displayName: '入会昵称',
      // 会议号
      meetingNum: '123456',
    }

    meetingService.joinMeeting(param)
    ```

    :::note note
    - 巡检者入会后只能切换布局视图，无法操作音视频和共享等。

    - 其他端无法感知到巡检者入会。
    :::

3. 解绑巡检者角色。

    当巡检者离开会议或者会议结束时，调用服务端的 [解绑巡检者角色](https://doc.yunxin.163.com/meeting/server-apis/DcwNDM3NjA) 接口进行角色解绑。
