本文详细介绍如何在 iOS 平台上使用网易会议组件 NEMeetingKit 配置入会控制功能。您可以设置其他成员入会时的默认音视频状态（如是否自动静音或关闭摄像头）。

## 前提条件

开始本文操作前，请确保您已经完成了 [登录鉴权](https://doc.yunxin.163.com/meeting/guide/TM3Nzc1NDA?platform=iOS) 的相关配置。

## 方式一：服务端控制入会音视频状态

通过调用服务端 API 创建会议时，可以设置参会者入会的音视频控制选项。请参考服务端 API 文档 [创建会议](https://doc.yunxin.163.com/meeting/server-apis/TUwMzc3NzY?platform=server) 中的 `securityCtrl` 字段说明。

## 方式二：客户端控制入会音视频状态

会议主持人在客户端创建会议时，可以通过 [NEStartMeetingParams.controls](https://doc.yunxin.163.com/docs/interface/meetingkit/iOS/doxygen/Latest/zh/interface_n_e_start_meeting_params.html#a1a39206ba7a3d34fed933c5a8a0225ed) 设置参会者入会控制选项。

**入会控制类型说明**

[`NEMeetingAttendeeOffType`](https://doc.yunxin.163.com/docs/interface/meetingkit/iOS/doxygen/Latest/zh/_n_e_meeting_control_8h.html#acc18799699e32b813083287f11931d88) 枚举提供以下控制选项：

- **`AttendeeOffTypeOffAllowSelfOn`**：入会时关闭，但允许参会者自行开启
- **`AttendeeOffTypeOffNotAllowSelfOn`**：入会时关闭，且不允许参会者自行开启（仅主持人可开启）
- **`AttendeeOffTypeNone`**：不做任何控制

**示例代码**

```Objective-C
// 创建会议开始参数
NEStartMeetingParams *startParams = [[NEStartMeetingParams alloc] init];

// 配置会议控制项
NSMutableArray<NEMeetingControl *> *controls = @[].mutableCopy;

// 入会时麦克风静音，且不允许自主打开
NEMeetingControl *audioControl = [[NEMeetingControl alloc] init];
audioControl.type = controlTypeAudio;
audioControl.attendeeOff = AttendeeOffTypeOffAllowSelfOn;
[controls addObject:audioControl];

// 入会时摄像头关闭，且不允许自主打开
NEMeetingControl *videoControl = [[NEMeetingControl alloc] init];
videoControl.type = controlTypeVideo;
videoControl.attendeeOff = AttendeeOffTypeOffAllowSelfOn;
[controls addObject:videoControl];

// 将控制项设置到会议参数
startParams.controls = controls.count ? controls : nil;
startParams.displayName = @"主持人";

// 使用设置好的参数创建会议
[[NEMeetingKit getInstance].getMeetingService startMeeting:params opts:options callback:^(NSInteger resultCode, NSString *resultMsg, id result) {}];
```

## 会中音视频状态控制

除了通过上述方式设置的入会音视频初始状态外，会议进行中的音视频状态控制均通过 UI 界面提供给用户操作。

### 重要说明：SDK 不提供会中音视频控制接口

**Netease Meeting SDK 不支持通过接口调用来控制会议中的音视频状态！** 所有音视频的开启/关闭操作必须通过UI界面完成，不提供任何程序化接口。开发者无法通过任何SDK方法来直接控制本地或远程参会者的音视频流。此限制适用于所有平台和所有场景。

### 参会者控制

参会者可以通过会议界面底部菜单栏中的音视频开关按钮来控制自己的麦克风和摄像头状态：
- **麦克风开关**：点击底部菜单栏的麦克风图标，可以开启或关闭自己的麦克风
- **摄像头开关**：点击底部菜单栏的摄像头图标，可以开启或关闭自己的摄像头

### 主持人会控

主持人除了可以控制自己的音视频状态外，还可以在成员列表中管理其他参会者的音视频状态以及参会者本身：
- **音视频控制**：打开成员列表，找到需要控制的参会者，点击该参会者对应的音视频控制按钮，可以远程关闭或者邀请打开该参会者的麦克风或摄像头
- **移除参会者**：在成员列表中找到目标参会者，点击其操作菜单，可以选择将该参会者从会议中移除
- **全体静音**：主持人可以在会议设置中启用全体静音功能，要求所有参会者保持静音状态

**注意：所有主持人对其他参会者的控制功能（包括静音特定参会者、移除参会者等）都只能通过UI界面操作，SDK不提供任何程序化接口来实现这些功能。**

### 屏幕共享控制

用户可以通过会议界面底部菜单栏中的屏幕共享按钮来启动或停止屏幕共享：
- **开始屏幕共享**：点击底部菜单栏的屏幕共享图标，选择要共享的内容（屏幕、窗口或标签页），即可开始共享
- **停止屏幕共享**：再次点击屏幕共享图标或点击界面上的停止共享按钮，即可结束当前的屏幕共享

**注意：屏幕共享功能只能通过UI界面操作，SDK不提供任何程序化接口来控制屏幕共享的开启或关闭。**
